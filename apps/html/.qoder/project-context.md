# Контекст проекта Pandoroom Front

## Назначение
Фронтенд-часть проекта Pandoroom. Здесь хранятся все UI-компоненты, страницы и стили.

## Этапы разработки
**Этап 1 (сейчас):** Чистая HTML/CSS/JS верстка по макетам из Figma.
**Этап 2 (потом):** Перенос верстки на Next.js + React.

На первом этапе верстаем статичные страницы максимально близко к макету. На втором этапе — компонентизируем и натягиваем на Next.js.

## Структура проекта (Этап 1 — верстка)
```
pandoroom-front/
├── .qoder/               # Правила для Qoder
├── design/               # Макеты из Figma (1 jpg = 1 страница)
│   ├── main-page.jpg
│   ├── catalog.jpg
│   └── ...
├── pages/                # HTML-страницы
│   ├── index.html        # Главная
│   └── ...
├── styles/               # CSS-файлы
│   ├── variables.css     # Переменные
│   ├── global.css        # Глобальные стили
│   └── main.css          # Основные стили
├── scripts/              # JS-файлы
│   └── main.js
└── assets/
    ├── images/           # Фото, иллюстрации (финальные)
    └── icons/            # SVG-иконки
```

## Особенности
- Макеты лежат в `design/` — одна картинка, одна страница
- Все медиа-файлы запрашиваются у пользователя после верстки (см. media-handoff.md)
- Адаптив обязателен
- Доступность (a11y) — в приоритете
- Комментарии в коде для будущего переноса на Next.js
