# Правила работы с медиа-ресурсами

## Главное правило
Если в макете есть изображения, иконки, SVG, видео или другие медиа — **не пытайся их подобрать самостоятельно**. Запроси их у пользователя после завершения верстки.

## Порядок работы

### 1. При верстке
- Для каждого медиа-элемента создай правильную разметку и стили
- Вместо `src` укажи путь по шаблону: `/assets/images/section-name/image-name.{ext}`
- Добавь комментарий рядом: `<!-- MEDIA_REQUEST: описание нужного файла -->`
- Если это фоновое изображение — оставь CSS-переменную или комментарий

### 2. После верстки
- Составь список всех необходимых медиа-файлов
- Укажи: название, формат, размеры, расположение в макете
- Попроси пользователя: "Вот список медиа, который нужен. Пришли файлы — я вставлю"

### 3. После получения медиа
- Размести файлы по папкам:
  - `/public/assets/images/` — фото, иллюстрации
  - `/public/assets/icons/` — иконки
  - `/public/assets/video/` — видео
  - `/src/assets/` — inline SVG, мелкая графика
- Обнови все пути в коде
- Проверь retina (@2x) если требуется

## Формат запроса медиа (пример)
```
Нужны следующие файлы:
1. hero-bg.jpg — фон первого экрана, ~1920x1080
2. product-1.png — карточка товара, ~400x400, прозрачный фон
3. icon-arrow.svg — стрелка в кнопке, ~24x24
4. logo-white.svg — логотип в футере
```

## SVG
- Простые иконки — inline SVG в коде (через React-компоненты)
- Сложные иллюстрации — отдельные .svg файлы
- Оптимизировать через svgo перед использованием
- Удалить width/height из атрибутов, контролировать размер через CSS

## Именование файлов
- kebab-case: `hero-background.jpg`, `icon-close.svg`
- Для retina: `image@2x.png`
- Не использовать русские буквы, пробелы, спецсимволы
