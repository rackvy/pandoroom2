# Pandoroom Front — Правила верстки для Qoder

## Главный принцип
Верстать **максимально 1 в 1** с макетом. Pixel-perfect — не просто слова, а стандарт.

## Технологии (Этап 1 — верстка)
- Чистый HTML5 + CSS3 + JS (vanilla)
- SVG — inline для иконок, отдельные файлы для сложной графики
- Шрифты — только те, что указаны в макете (Google Fonts / локальные)
- CSS-переменные в отдельном файле `styles/variables.css`

## Технологии (Этап 2 — Next.js)
- Будет перенесено позже. Сейчас верстаем так, чтобы легко было декомпозировать на React-компоненты.

## Методология
- БЭМ для именования классов (или аналог в CSS Modules)
- Компонентный подход: каждый блок макета — отдельный компонент
- Семантический HTML5 (`header`, `nav`, `section`, `article`, `footer`)

## Адаптив
- Mobile-first подход
- Брейкпоинты: 640px, 768px, 1024px, 1280px, 1536px
- На мобильных проверять тач-зоны (минимум 44x44px)

## Качество кода
- CSS-переменные для всех цветов, отступов, скруглений, теней
- Типографика через переменные (размер, вес, интерлиньяж)
- Доступность: alt-тексты, aria-labels, focus-стили
- Не оставлять `console.log` и закомментированный код

## Запрещено
- Использовать `!important` без крайней необходимости
- Писать магические числа (всё через переменные)
- Использовать inline-стили (кроме динамических значений в React)
- Оставлять placeholder тексты типа "Lorem ipsum" — либо реальный контент, либо `[ТЕКСТ]`

## Медиа-ресурсы
- **НЕ** подставлять случайные картинки с интернета
- **НЕ** использовать сервисы-заглушки (placeholder.com и т.п.)
- Смотри `media-handoff.md` — там правила работы с медиа
